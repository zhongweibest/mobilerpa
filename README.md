# MobileRPA

## 协作约定

### 1. 文档语言

默认情况下，项目文档、开发说明、验收文档和代码注释均使用中文。

### 2. 文档编号顺序

所有正式文档中的章节编号、步骤编号和验收编号都必须保持连续、顺序正确。

要求如下：

1. 不能出现跳号
2. 不能把后续新增章节追加到错误位置导致顺序错乱
3. 修改文档后，如果新增了 `4.9`，它必须紧跟在 `4.8` 之后
4. 如果新增内容影响原有编号，需要同步整理整份文档的编号

### 3. Git 提交信息

默认情况下，Git 提交信息使用中文。

要求如下：

1. 提交信息应简洁明确
2. 优先描述本次提交的主要变更
3. 非必要不使用英文提交信息

### 4. 计划与实现顺序

新增会影响整体方案、阶段计划、项目看板、协议、配置或任务编排方式的功能时，先更新文档，再进入实现。

至少同步以下文档：

1. `docs/整体规划方案.md`
2. `docs/开发阶段计划.md`
3. `docs/项目看板.md`
