# S1 阶段验收记录

## 1. 阶段信息

- 阶段编号：`S1`
- 阶段名称：中心接入主链路
- 记录用途：沉淀 `S1` 阶段的人工验收结论、已通过项、待后续阶段继续增强项

## 2. 阶段目标

`S1` 阶段目标如下：

1. 设备可以注册
2. 中心可以识别设备
3. Agent 可以注册并保持心跳
4. 中心可以通过 WebSocket 维护基础在线状态
5. 前端设备列表页可以展示真实设备状态字段

## 3. 已完成周期

本阶段已完成并已由用户明确验收通过的周期：

1. `C1-1` WebSocket `hello/heartbeat` 链路
2. `C1-1A` 心跳超时自动离线增强
3. `C1-2` Agent 注册与心跳
4. `C1-3` 设备列表前端入口

## 4. 已通过的验收结论

### 4.1 中心接入链路

- 中心服务可启动
- 设备可通过 HTTP 注册
- 设备列表和单设备详情可查询
- `hello/heartbeat` 可建立在线状态
- 断线或超时后设备可自动变为离线

### 4.2 Agent 链路

- `agent.js` 可在真机侧启动
- Agent 可生成并保存 `agent_uuid`
- Agent 可注册并保存 `device_id`
- Agent 可连接中心 `/ws`
- Agent 可稳定发送心跳

### 4.3 前端设备页

- 前端页面可打开
- 设备列表接口可接入
- 页面可展示 `status`
- 页面可展示 `bind_status`
- 页面可展示 `last_heartbeat_at`
- 刷新按钮可用
- 本地联调不存在跨域阻塞
