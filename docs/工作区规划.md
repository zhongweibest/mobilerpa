# MobileRPA 工作区规划

> 维护说明：本文档只作为工作区结构入口说明。详细整体方案以 `docs/整体规划方案.md` 为准，当前执行状态以 `docs/项目看板.md` 为准。

## 目标
当前工作区已经按“多仓库协作工作区”组织，支持以下能力：

- 基于 Go 的中心控制服务
- 基于 Vue 的后台管理前端
- AutoJs 手机端 Agent 与业务脚本
- 独立的真机快速调试链路

## 仓库策略
当前在同一个本地工作区内维护多个独立仓库：

1. `repos/mobilerpa-center`
   - 当前采用 monorepo 形式，包含 Go 中心服务后端和 Vue 管理后台前端。
   - 管理后台是中心端的 Web 管理入口，不是独立于中心服务之外的另一套业务中心。

2. `repos/mobilerpa-agent`
   - 包含手机端 Agent 运行仓库，内部区分 `js-agent/` 回滚基线与 `ts-agent/` 正式维护主线。

3. `repos/mobilerpa-script`
   - 包含业务脚本源码工程、共享 helper 和发布产物目录。

4. `repos/mobilerpa-toolkit`
   - 包含设备发现、Agent 推送、启动与停止等独立工具能力。

## Git 版本管理规划

当前工作区采用“四个 Git 仓库 + 一个根文档仓”的结构：

1. 工作区根仓库：`D:\dev\code\mobilerpa`
   - 用于管理工作区级文档、规划、看板、归档、通用说明和安全的配置模板。
   - 默认纳入：`docs/`、根目录 `README.md`、根目录 `.gitignore`、可复用的工作区脚本模板。
   - 默认不纳入：`projects/`、`scripts/`、`.tmp/`、本机真实配置、运行数据库、构建产物、设备私有配置。

2. 中心端仓库：`repos/mobilerpa-center`
   - 继续作为独立 Git 仓库。
   - 管理 Go 中心服务后端、Vue 管理后台前端，以及中心端模块文档和验收文档。

3. Agent 仓库：`repos/mobilerpa-agent`
   - 继续作为独立 Git 仓库。
   - 管理手机端 Agent、迁移方案、运行契约和验收文档。

4. 脚本仓库：`repos/mobilerpa-script`
   - 继续作为独立 Git 仓库。
   - 管理业务脚本 TypeScript 源码工程、共享运行时能力和发布产物目录。

5. 工具仓库：`repos/mobilerpa-toolkit`
   - 继续作为独立 Git 仓库。
   - 管理设备发现、Agent 推送、启动与停止等跨平台工具。

这种结构的好处：

1. 根目录文档和规划有 Git 兜底，移动、归档或删除文档都可以安全追溯。
2. 中心服务与 Agent 保持独立提交历史，避免两个运行端互相污染版本节奏。
3. `projects/`、`scripts/` 继续作为 legacy 源材料，不因临时脚本、历史脚本和本机调试文件污染正式仓库。
4. 根仓库只维护文档和工作区说明，不直接承载子仓库业务代码。

### 根仓库默认忽略规则

根目录 `.gitignore` 至少包含：

```text
.tmp/
projects/
scripts/
config.ini
choose_device_push.bat

.vscode/
!.vscode/*.example.json

repos/mobilerpa-center/
repos/mobilerpa-agent/
repos/mobilerpa-script/
repos/mobilerpa-toolkit/
```

说明：

1. `.tmp/` 是运行产物和测试数据库，不进 Git。
2. `projects/`、`scripts/` 是旧脚本源材料，先不进 Git。
3. `config.ini` 和 `choose_device_push.bat` 可能包含本机路径、设备序列号或私人配置，先不进 Git。
4. `.vscode/` 当前包含真实设备序列号，默认只提交 `.example.json` 模板。
5. `repos/` 下各子仓库由各自 Git 管理，根仓库默认忽略它们，避免嵌套仓库状态混乱。

## 为什么前端不单独拆仓库
前端单独拆仓库通常适用于以下情况：

- 前端团队独立运作
- 前后端发布节奏明显不同
- API 协议已经相对稳定

当前并不采用这种拆分方式。管理后台与中心服务后端围绕同一套协议、设备模型和发布流程协同演进，因此中心端仓库继续采用 monorepo 结构。

## Legacy 代码处理原则
当前的 `projects/` 和 `scripts/` 等目录仍视为 legacy 源材料。
它们不作为正式主线代码目录；正式 Agent 维护在 `repos/mobilerpa-agent`，正式业务脚本维护在 `repos/mobilerpa-script`。

## 文档语言约定
- 当前工作区内的项目文档默认使用中文编写。
- 新增的架构、开发、协议、部署、运维文档，默认也使用中文。
- 新增的代码注释默认使用中文编写，包括方法注释、结构体注释、字段注释和必要的行内注释。
- 如确有必要引入英文内容，应以中文为主、英文为辅。

## 当前工作区定位
1. 根目录用于维护总纲、看板、阶段计划、验收记录和文档治理规则。
2. `repos/mobilerpa-center` 维护中心端能力。
3. `repos/mobilerpa-agent` 维护手机端 Agent。
4. `repos/mobilerpa-script` 维护业务脚本源码与发布产物。
5. `repos/mobilerpa-toolkit` 维护设备发现和 Agent 运维工具。
