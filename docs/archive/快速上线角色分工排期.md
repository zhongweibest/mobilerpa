# MobileRPA 快速上线角色分工排期

> 归档说明：本文档已归档，只作为早期角色分工排期讨论记录。当前角色边界以 `docs/开发阶段计划.md` 和 `docs/项目看板.md` 为准。

> 维护说明：本文档保留为早期角色分工排期参考。当前执行状态、任务完成情况和下一步优先级以 `docs/项目看板.md` 为准；若两者不一致，优先采用看板。

## 1. 文档目的
本文档将快速上线计划进一步拆成按角色分工的执行排期，目标是让团队可以直接按角色开工，并支持并行推进。

本文档默认目标为：

1. 4 周内做出早期生产版
2. 第一个投产目标为 10 到 30 台设备
3. 后续继续向 50 到 100 台、再到 300 台扩容

## 2. 角色定义
推荐按以下 4 个角色推进：

1. 后端角色
2. Agent/脚本角色
3. 前端角色
4. 联调与验收角色

如果团队规模较小，可以兼任，但责任边界尽量保持清晰。

### 2.1 后端角色
负责范围：

- `repos/mobilerpa-center/server`
- 中心服务配置、SQLite、HTTP、WebSocket、任务主链路
- 文件接收和最小日志归档

### 2.2 Agent/脚本角色
负责范围：

- `repos/mobilerpa-agent/agent`
- `repos/mobilerpa-agent/scripts`
- `repos/mobilerpa-agent/tools`

### 2.3 前端角色
负责范围：

- `repos/mobilerpa-center/web`
- 后台页面、任务下发入口、设备查看与基础绑定页面

### 2.4 联调与验收角色
负责范围：

- `docs/acceptance-checklists`
- 测试机准备
- 手工验收步骤执行
- 问题回归与验收记录

## 3. 角色协作原则

### 3.1 串行依赖
以下依赖必须尽量先满足：

1. 后端先给出注册接口与 WebSocket 协议第一版
2. Agent 再接注册和长连接
3. Agent 先有最小任务执行骨架
4. 前端再接设备页和任务页接口

### 3.2 并行原则
以下工作可以同步推进：

1. 后端做数据库和接口
2. Agent/脚本做本地身份、长连接和脚本入口
3. 前端做页面骨架和接口占位
4. 联调角色做验收清单、测试环境和试运行准备

### 3.3 验收原则
每周都必须有人工功能验收，不允许只靠代码完成度判断进度。

## 4. 第 1 周排期

## 4.1 后端角色
本周目标：设备能注册，中心能识别设备。

任务项：

1. 完成 SQLite 表结构第一版
2. 完成数据库初始化入口
3. 完成 `devices`、`tasks`、`task_events` 等核心表
4. 完成设备注册接口
5. 完成 `device_id` 分配规则
6. 完成设备基础信息写入
7. 完成 WebSocket 接入入口
8. 完成 `hello` 和 `heartbeat` 基础消息处理
9. 完成设备列表查询接口

本周产出：

1. 可启动的中心服务
2. 可初始化的 SQLite
3. 可注册与可上线的设备接入链路

### 4.2 Agent/脚本角色
本周目标：Agent 能准备好接入中心。

任务项：

1. 建立 `agent.js` 主入口骨架
2. 建立本地配置文件结构
3. 实现 `agent_uuid` 生成与持久化
4. 实现 `device_id` 本地保存占位
5. 建立本地目录初始化逻辑
6. 确认 `shoppe` 脚本迁移边界
7. 确认 `src/` 与 `debug/` 拆分方案

本周产出：

1. Agent 基础骨架
2. 脚本迁移清单

### 4.3 前端角色
本周目标：准备后台工程骨架。

任务项：

1. 确认前端技术栈与工程结构
2. 搭建设备页和任务页的最小框架
3. 准备 API SDK 基础封装
4. 准备通用布局和导航
5. 明确设备列表、任务列表所需字段

本周产出：

1. 前端工程起盘方案
2. 页面骨架和接口占位

### 4.4 联调与验收角色
本周目标：准备接入验收环境。

任务项：

1. 准备 1 到 3 台测试机
2. 准备中心服务联调地址
3. 整理设备注册验收清单
4. 整理心跳与重连验收清单
5. 记录首次接入步骤

本周验收重点：

1. 设备注册
2. 设备上线
3. 在线状态可见

## 5. 第 2 周排期

## 5.1 后端角色
本周目标：中心可以向设备下发一个最小任务。

任务项：

1. 完成基础任务模型
2. 完成任务创建接口
3. 完成单设备任务下发逻辑
4. 完成任务状态回写
5. 完成基础结果接收接口或消息处理
6. 完成基础日志接收

本周产出：

1. 单任务下发主链路
2. 平台侧任务状态记录

### 5.2 Agent/脚本角色
本周目标：设备收到任务后能拉起脚本。

任务项：

1. 完成注册调用
2. 完成 WebSocket 长连接
3. 完成心跳发送
4. 完成任务消息接收
5. 完成任务执行入口
6. 完成执行结果回传
7. 完成基础日志回传
8. 完成 `shoppe` 第一版主入口迁移

本周产出：

1. 真机可从中心接任务
2. 真机可执行脚本入口并回传结果

### 5.3 前端角色
本周目标：能看到设备和任务基础状态。

任务项：

1. 接设备列表接口
2. 接任务列表接口
3. 完成设备页基础展示
4. 完成任务页基础展示
5. 预留任务创建入口

本周产出：

1. 设备页
2. 任务页

### 5.4 联调与验收角色
本周目标：完成单机到单任务联调。

任务项：

1. 执行任务下发与脚本执行验收
2. 执行任务进度与结果回传验收
3. 记录脚本执行失败样例
4. 整理高频联调问题

本周验收重点：

1. 单任务下发
2. 脚本执行
3. 结果回传

## 6. 第 3 周排期

1. 完成日志保存最小链路

4. 完成 `physical_slot` 基础字段写入

1. 基础日志与截图可回看

### 6.2 Agent/脚本角色

任务项：

1. 迁移关键页面模块

3. 增加关键步骤进度上报
4. 建立本地调试入口

本周产出：

1. 第一版可联调的 `shoppe`

### 6.3 前端角色
本周目标：让后台具备最小操作闭环。

任务项：

1. 完成任务创建基础表单
2. 完成设备绑定基础表单
3. 完成日志和结果查看入口
4. 优化设备页与任务页基础交互

本周产出：

### 6.4 联调与验收角色

任务项：

1. 执行日志与截图上传验收

3. 组织 3 台到 10 台设备联调
4. 记录脚本稳定性问题
5. 记录连接稳定性问题

1. 日志与截图

## 8. 角色并行工作矩阵

### 8.1 后端与 Agent
可以并行：

1. 后端定义注册接口
2. Agent 实现注册调用
3. 后端定义 WebSocket 消息
4. Agent 实现消息收发

### 8.2 Agent 与脚本
可以并行：

1. Agent 做任务装配
2. 脚本做 `shoppe` 迁移
3. 通过稳定入口对接

### 8.3 前端与后端
可以并行：

1. 后端先给字段和接口约定
2. 前端先用 mock 或占位接口起页面骨架
3. 后续再接真实接口

### 8.4 联调与其他角色
联调角色从第 1 周就应该介入，不应等开发做完再开始。

## 9. 当前执行建议
从当前状态开始，建议马上分三条主线开工：

1. 后端主线：SQLite 初始化、设备注册、WebSocket 接入
2. Agent 主线：`agent.js` 骨架、注册调用、长连接
3. 前端主线：设备页和任务页骨架

联调与验收主线同步开始准备测试机和验收记录。

按这个排期推进，4 周内最有机会做出一个可以小规模投入生产的版本。
