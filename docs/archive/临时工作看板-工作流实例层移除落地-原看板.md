# 临时工作看板：工作流实例层移除落地

> 维护说明：本文档是本轮“工作流实例层移除与执行域合并”专项改造的临时执行看板。
> 正式周期状态仍以 `docs/项目看板.md` 为准；本文件用于集中跟踪本轮差距、落地顺序、风险和验证点。

## 1. 目标

将工作流型计划任务的运行态从旧的 `workflow_instances` / `workflow_runs` 彻底收口到：

```text
plan_def
  -> plan_run
     -> plan_device_run
```

本轮落地后应满足：

1. 启动工作流型计划任务时，不再创建 `workflow_instances` / `workflow_runs`。
2. 运行态控制统一归计划任务实例，不再存在独立工作流实例运行域。
3. 设备占用统一只看 `plan_device_runs.status in (pending, running)`。
4. `internal/workflow` 只保留定义与快照构建，不再承担运行实例生命周期。
5. 前端不再依赖“工作流实例”页或相关实例接口。
