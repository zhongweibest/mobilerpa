# 日志与截图上传验收清单

## 验收目标
确认设备执行中的日志和截图能够上传、归档并用于人工排障。

## 前置条件
1. Agent 已具备日志回传能力
2. 平台已具备日志接收或文件上传能力
3. 脚本执行过程中可产生截图

## 验收步骤
1. 下发一个测试任务
2. 观察脚本执行过程中是否产生关键日志
3. 触发一个可预期异常，强制生成截图
4. 检查平台是否收到日志
5. 检查平台或文件目录中是否存在目标截图
6. 检查文件命名和归档路径是否符合规范
7. 用日志和截图回放问题

## 预期结果
1. 日志可查看
2. 截图可查看
3. 日志和截图能关联到具体设备和任务
4. 文件路径清晰，不混乱

## 异常验证
1. 上传失败时是否有重试或错误日志
2. 大量日志时平台是否还能看关键内容
3. 截图路径错误时是否可定位

## 验收结论
1. 通过
2. 有条件通过
3. 不通过
