# 任务进度与结果回传验收清单

## 验收目标
确认脚本执行过程中的关键步骤、结果和错误能被平台正确接收和展示。

## 前置条件
1. 已有状态机式脚本步骤
2. 平台支持基础任务状态查看

## 验收步骤
1. 下发一条测试任务
2. 观察脚本每个关键步骤是否上报 `task_progress`
3. 核对步骤顺序是否符合预期
4. 人工制造一个失败场景
5. 观察平台是否收到失败结果
6. 检查失败信息是否可定位问题
7. 再执行一次成功场景
8. 检查平台是否收到成功结果

## 预期结果
1. 关键步骤有清晰进度
2. 成功与失败结果可区分
3. 错误信息具备定位价值
4. 平台展示与真实执行一致

## 异常验证
1. 漏报某一步时平台表现如何
2. 结果重复回传时平台是否异常
3. 中途退出时任务状态是否可解释

## 验收结论
1. 通过
2. 有条件通过
3. 不通过
