# 心跳与断线重连验收清单

## 验收目标
确认设备长连接、心跳和断线重连在真实网络波动下可用。

## 前置条件
1. 设备已完成注册
2. WebSocket 接入已可用
3. 设备页或接口能查看在线状态

## 验收步骤
1. 启动 Agent 并建立 WebSocket 连接
2. 观察中心是否收到 `hello`
3. 观察中心是否持续收到 `heartbeat`
4. 在后台确认设备为在线状态
5. 人工断开设备网络
6. 观察中心是否在预期时间内标记离线
7. 恢复设备网络
8. 观察 Agent 是否自动重连
9. 观察中心是否恢复在线状态
10. 重复以上过程至少 3 次

## Apifox 手工验收示例

### 1. 注册测试设备

请求方法：

```text
POST
```

请求地址：

```text
http://127.0.0.1:18080/api/v1/device/register
```

请求头：

```text
Content-Type: application/json
```

请求体：

```json
{
  "agent_uuid": "agent-test-001",
  "device_name": "Pixel Test",
  "brand": "Google",
  "model": "Pixel 8",
  "android_id": "android-test-001",
  "adb_serial": "adb-test-001"
}
```

记录响应中的 `data.device_id`，后续示例假设它是 `dev_000001`。

### 2. 建立 WebSocket 连接

连接地址：

```text
ws://127.0.0.1:18080/ws
```

### 3. 发送 `hello`

```json
{
  "type": "hello",
  "request_id": "apifox-hello-001",
  "device_id": "dev_000001",
  "timestamp": 1780935112,
  "payload": {
    "agent_uuid": "agent-test-001"
  }
}
```

预期响应：

```json
{
  "type": "ack",
  "request_id": "apifox-hello-001",
  "device_id": "",
  "timestamp": 1780935112,
  "payload": {
    "message_type": "hello",
    "status": "ok"
  }
}
```

### 4. 发送 `heartbeat`

```json
{
  "type": "heartbeat",
  "request_id": "apifox-heartbeat-001",
  "device_id": "dev_000001",
  "timestamp": 1780935113,
  "payload": {
    "battery": 90,
    "network": "wifi"
  }
}
```

预期响应：

```json
{
  "type": "ack",
  "request_id": "apifox-heartbeat-001",
  "device_id": "",
  "timestamp": 1780935113,
  "payload": {
    "message_type": "heartbeat",
    "status": "ok"
  }
}
```

### 5. 查询设备状态

请求：

```text
GET http://127.0.0.1:18080/api/v1/devices
```

预期结果：

1. `status` 为 `online`
2. `last_heartbeat_at` 不为空

### 6. 断开连接并复查

在 Apifox 中断开 WebSocket，再调用设备列表接口。

预期结果：

1. `status` 变为 `offline`
2. `last_heartbeat_at` 保留最近一次心跳时间

## 预期结果
1. 心跳稳定发送
2. 离线状态能被正确识别
3. 网络恢复后能自动重连
4. 重连后仍为同一设备身份

## 异常验证
1. 快速反复断网时是否出现假死连接
2. 中心重启后 Agent 是否能恢复连接
3. 重连失败时日志是否可定位
4. 如果返回 `invalid_json`，检查是否发送了合法 JSON 文本，不能发送 PowerShell 的 `@{}` 格式

## 验收结论
1. 通过
2. 有条件通过
3. 不通过
