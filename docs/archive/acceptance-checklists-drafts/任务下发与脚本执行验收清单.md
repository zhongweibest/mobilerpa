# 任务下发与脚本执行验收清单

## 验收目标
确认平台能够向指定设备下发任务，设备能够拉起脚本并回传结果。

## 前置条件
1. 设备已在线
2. Agent 已具备任务接收能力
3. 已有一个可执行脚本入口

## 验收步骤
1. 在后台或接口创建一个测试任务
2. 将任务下发给指定测试设备
3. 观察中心是否记录任务创建
4. 观察 Agent 是否收到任务消息
5. 观察 Agent 是否拉起目标脚本入口
6. 观察脚本执行过程中的关键日志
7. 观察平台是否收到结果回传
8. 观察任务状态是否变为成功或失败

## 预期结果
1. 任务能到达指定设备
2. 设备只执行一次目标任务
3. 平台能看到任务状态变化
4. 执行结束后有明确结果

## 异常验证
1. 设备离线时下发任务行为是否符合预期
2. 脚本执行失败时是否能回传错误
3. 重复下发同一任务是否会出现重复执行

## 验收结论
1. 通过
2. 有条件通过
3. 不通过
