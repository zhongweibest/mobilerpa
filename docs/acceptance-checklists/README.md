# MobileRPA 验收入口索引

本目录只作为项目级验收入口索引，不再存放具体功能验收清单。

当前验收文档按“两层入口”维护：

1. 中心端正式页面与业务主线，统一维护在根目录文档。
2. 手机端与脚本、中心服务底层排查，继续维护在各模块自己的 `docs/` 目录中。

## 验收文档原则

1. 只为已经实现或正在等待用户验收的功能提供可执行验收文档。
2. 未实现功能不提前编写完整验收清单，只在对应设计文档中保留验收目标或能力边界说明。
3. 页面主线验收文档以真实页面操作顺序组织，必要时补充底层接口或日志检查点。
4. 服务级补充检查文档必须提供真实启动方式、关键接口地址和最小排查步骤。
5. 单元测试、脚本自测或我自行判断通过，不能替代用户人工验收通过。
6. 功能完成实现后，如果原有验收文档不存在，必须先补正式验收文档，再交给用户验收。

## 如何选择验收文档

选择顺序：

1. 先确认要验收的是中心端页面主线，还是手机端/服务端补充链路。
2. 中心端正式运行主线优先查看 `docs/页面与主线手工验收.md`。
3. 如果页面结果异常，再查看对应模块的补充检查文档。
4. 如果还没有真实可执行验收文档，不能按草稿清单验收，应先补正式验收文档。

## 当前可执行验收入口

| 模块 | 覆盖能力 | 可执行验收文档 | 说明 |
|---|---|---|---|
| 中心端 | 后台页面、设备发现、设备绑定、脚本管理、工作流定义、计划任务与计划任务实例主线 | `docs/页面与主线手工验收.md` | 正式中心端验收入口 |
| 中心服务补充检查 | 服务启动、健康检查、设备注册、WebSocket 烟测与页面异常排查 | `repos/mobilerpa-center/docs/server/中心服务补充检查手册.md` | 页面主线异常时的补充排查入口 |
| 手机端 Agent | 注册、心跳、脚本下载、脚本执行、停止与会话执行 | `repos/mobilerpa-agent/docs/手机端手工验收.md` | 正式手机端验收入口 |

## 验收文档补充原则

- 中心端正式页面主线优先补到根目录验收文档，而不是同时在 `server` 和 `web` 各写一份。
- 服务级、设备级、脚本级补充排查文档继续保留在对应模块 `docs/` 目录中。
- 阶段性结论、试运行结论和专项验收结果继续放在 `docs/验收记录/`。
- 不再维护按开发周期拆分的项目级验收入口。
