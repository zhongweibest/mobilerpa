# 新脚本接入模板清单

为避免每个业务脚本各写一套结构，新增脚本时应直接在 `repos/mobilerpa-script/packages/` 下创建独立脚本包，并通过统一发布流程生成构建产物。

## 目录模板

```text
packages/
  <script-package>/
    src/
      index.ts
    package.json
    tsconfig.json
    script.config.json
```

## 创建新脚本时必改项

- `script.config.json` 中的脚本名、版本号和发布配置
- 源码中的 `SCRIPT_VERSION` 或等价版本常量
- `src/index.ts` 中的步骤名、结果码和业务返回数据
- 调试入口中的默认调试参数

## 接入完成前自检

- 本地 `index_debug.js` 入口可以独立运行
- `run(context)` 返回结构满足 Agent 契约
- 失败分支会返回明确结果码，而不是只抛模糊异常
- 关键步骤已预留日志点
- 需要截图或 dump 的关键失败点已留扩展位

## 推荐接入顺序

1. 先用占位逻辑把上下文和结果结构跑通
2. 再补页面识别和步骤流转
3. 再接入共享 helper，例如应用启动、取消检查、日志与公共 UI 操作
4. 最后补异常处理、截图和重试策略
