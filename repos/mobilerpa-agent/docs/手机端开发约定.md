# Agent 开发约定

## 脚本结构约定
- `src/` 只放可复用的业务逻辑
- `debug/` 只放有明确目的的调试入口
- 不要把平台相关网络代码直接塞进页面模块或流程模块

## 版本流转约定
- `dev`：开发者本地验证版本
- `beta`：通过中心平台在测试设备组验证的版本
- `prod`：正式生产发布版本

## 调试流程约定
- 先在本地单步或单流程调通
- 再通过中心平台把相同版本下发到调试设备验证
- 只有步骤级和任务级验证都通过后，才允许继续升级版本

## 身份与配置约定
- Agent 本地身份优先使用已保存的 `agent_uuid`
- 只有本地没有 `agent_uuid` 时，才允许根据设备稳定指纹生成新的 `agent_uuid`
- 推送 Agent 代码时，默认不覆盖手机端已有 `runtime/config.json`
- 推送 Agent 代码时，默认应刷新手机端 `runtime/bootstrap.json`，以同步最新的 `center_base_url` 和连接参数
- Agent 启动时必须保留已有 `agent_uuid` / `device_id`，但允许用 `bootstrap.json` 更新中心地址
- 只有明确执行重置操作时，才允许清空或重建本地 `agent_uuid` / `device_id`
- 生产环境与中心服务的正式推送、启动、停止路径默认只维护 `mobilerpa-toolkit`
- `tools/adb-debug` 下的 `.bat`、`.ps1` 仅作为本地开发调试兜底，不作为生产依赖和正式验收主路径

## 日志约定
- `debug` 模式允许输出详细日志，并在需要时自动截图或导出 dump
- `prod` 模式只记录任务关键日志，避免体积失控

## 长连接约定
- Agent 与中心的 WebSocket 长连接默认开启自动重连
- 默认重连节奏为 `3s -> 6s -> 12s -> 24s -> 48s -> 60s -> 60s ...`
- 到达 `60s` 后继续以 `60s` 为固定间隔无限次重试，直到中心恢复或 Agent 被人工停止
- 每次重连成功后，必须重新发送 `hello`，收到 `hello ack` 后才恢复心跳
- 任意时刻只允许存在一个连接实例、一个心跳调度器和一个重连调度器

## 停止机制约定
- “停止 Agent”默认指停止 `agent.js` 本身，而不是强制退出整个 AutoJs6
- 设备侧停止信号文件固定为 `/sdcard/脚本/agent/runtime/stop.signal`
- 运行中的 `agent.js` 检测到 `stop.signal` 后，必须先关闭 WebSocket、停止心跳、停止重连，再自行退出
- 启动 Agent 前，启动入口必须先清理旧的 `stop.signal`，避免新启动后立即退出
- 除非用户明确要求强制停止整个 AutoJs6，否则不允许把“停止 Agent”实现成 `force-stop org.autojs.autojs6`

## 文档约定
- 项目文档默认使用中文编写
- 代码注释默认使用中文编写，包括方法注释、结构体注释、对象字段注释和必要的行内注释
- 调试说明、接入说明、发布说明都应沉淀为中文记录
- 包名、变量名、字段名、接口路径、协议字段和第三方库名称保留其技术标识，不强行翻译
- 只要涉及 Agent 推送、启动、停止、批量下发、网页下发或生产操作，文档默认优先给出 `mobilerpa-toolkit` 的命令与步骤

## 协作确认约定
- 如果我对需求有疑问，或者觉得某个要求不合理，必须先停下来说明疑问、风险、建议方案和可选方案，等待用户确认后再继续。
- 涉及工程惯例、工具默认识别或平台展示的命名，例如 `README.md`、配置文件名、启动入口和目录结构，不能由我直接做最终决定。
- 计划顺序、开发周期、任务状态和验收结论的调整，必须先征得用户确认。
- 用户确认前，只允许进行阅读、检查、梳理和提出建议，不擅自执行会改变结果的操作。
