# 新脚本接入模板清单

为避免每个业务脚本各写一套结构，新增脚本时建议直接复制 `scripts/_template/` 并替换占位内容。

## 目录模板

```text
scripts/
  _template/
    manifest.json
    src/
      main.js
    debug/
      debug_main.js
```

## 创建新脚本时必改项

- `manifest.json` 中的 `script_name`
- `manifest.json` 中的 `version`
- `manifest.json` 中的 `description`
- `src/main.js` 中的步骤名、结果码和业务返回数据
- `debug/debug_main.js` 中的默认调试参数

## 接入完成前自检

- 本地 `debug` 入口可以独立运行
- `run(context)` 返回结构满足 Agent 契约
- 失败分支会返回明确结果码，而不是只抛模糊异常
- 关键步骤已预留日志点
- 需要截图或 dump 的关键失败点已留扩展位

## 推荐接入顺序

1. 先用占位逻辑把上下文和结果结构跑通
2. 再补页面识别和步骤流转
3. 最后补异常处理、截图和重试策略
