# Agent 当前运行方案

## 当前结论
- 当前 Agent 最终采用 `充电 + 不锁屏/常亮` 的运行策略。
- 不再默认使用 `AlarmManager` 心跳实验方案。
- 不再使用 `CPU WakeLock` 与 `Wi-Fi Lock` 保活。
- 保留已经验证有效的稳定性修复：
- 独立运行锁目录
- 更严格的多实例拦截
- 心跳租约机制
- 优雅停止与旧链路自停

## 当前默认行为

### 心跳
- 默认心跳调度模式为 `executor`。
- WebSocket 建连成功后立即发送一次 `hello_ack` 心跳。
- 后续按 `heartbeat_interval_ms` 周期持续发送心跳。

### 多实例拦截
- Agent 运行锁放在独立运行目录：
- `/sdcard/脚本/.mobilerpa-agent-runtime/agent.lock.json`
- 当旧实例未停止时，再次启动会直接返回 `already_running`，不会再重复注册和重复建连。

### 心跳租约
- 心跳租约文件放在独立运行目录：
- `/sdcard/脚本/.mobilerpa-agent-runtime/heartbeat.lease.json`
- 如果旧心跳线程因为历史原因仍被系统唤醒，会在发现租约失效后立即自停，不再继续发心跳。

### 停止
- 仍然通过 `stop.signal` 停止 Agent。
- 收到停止信号后，会关闭 WebSocket、停止心跳调度，并释放运行锁。

## 已移除内容
- `AlarmManager` 心跳调度相关实现已移除，不再作为默认或实验方案启用。
- `CPU 保活锁` 相关实现已移除。
- `Wi-Fi 保活锁` 相关实现已移除。
- 保活锁巡检逻辑已移除。

## 推荐运行方式
- 设备接入充电。
- 关闭锁屏或保持屏幕常亮。
- 尽量关闭系统省电限制，避免 AutoJs6 被系统强杀。

## 当前验收结论
- 心跳发送正常。
- 二次启动拦截正常。
- 停止后心跳收尾正常。
- 运行形态已经符合“无额外保活锁、无 AlarmManager、常亮运行”的最终方案。
