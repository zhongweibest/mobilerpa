# Agent 手工启动与注册验收

## 1. 目的

本文档用于验收 Agent 的最小启动和注册链路。

当前适用范围：

1. `agent.js` 主入口可执行。
2. Agent 可基于 `ANDROID_ID` 生成并保存 `agent_uuid`。
3. Agent 可调用中心注册接口。
4. Agent 可保存中心返回的 `device_id`。
5. Agent 可连接中心 WebSocket。
6. Agent 可发送 `hello` 和周期 `heartbeat`。
7. 中心服务短暂中断后，Agent 可自动重连并恢复心跳。

当前正式运行入口约定如下：

1. 正式运行主线统一收口到“计划任务 + 工作流”。
2. 工作流正式执行默认走 `workflow_session_*` 协议族，由中心下发工作流会话、由 Agent 本地推进状态机。
3. 工作流实例层已经移除，正式运行态统一为 `plan_def -> plan_run -> plan_device_run`。

当前暂不完整覆盖：

1. `shoppe` 真正业务脚本迁移。
2. 百分比进度展示。
3. 细粒度实时流式日志。
4. 脚本包下载与版本管理。

其中“计划任务实例指令接收、最小执行链路与结果回传”已经在本文档后续章节补充；剩余真实业务脚本链路会在 `C2-3` 后续迭代继续补充到本文档。

## 2. 前置条件

1. 中心服务已经启动。
2. 中心服务地址默认为 `http://127.0.0.1:18080`。
3. 本机已安装 Node.js，或已在手机上准备 AutoJs6 环境。

预期结果：

1. 不再生成新的 `agent_uuid`。

## 6. 真机 AutoJs6 验收

### 6.1 启动中心服务

真机访问电脑服务时，中心服务不能只监听 `127.0.0.1`。建议中心服务使用：

```powershell
cd D:\dev\code\mobilerpa\repos\mobilerpa-center\server
$env:CENTER_HTTP_ADDR="0.0.0.0:18080"
$env:CENTER_DB_PATH="D:\dev\code\mobilerpa\.tmp\mobilerpa-center-manual.db"
go run .\cmd\center
```

### 6.2 获取电脑局域网 IP

```powershell
Get-NetIPAddress -AddressFamily IPv4 |
  Where-Object { $_.IPAddress -notlike "127.*" -and $_.PrefixOrigin -ne "WellKnown" } |
  Select-Object IPAddress, InterfaceAlias
```

假设电脑 IP 是 `192.168.1.23`，真机访问中心服务时使用：

```text
http://192.168.1.23:18080
```

注意：真机上不能使用 `http://127.0.0.1:18080`。

### 6.3 推送并运行 Agent

确认 ADB 已连接真机：


### 6.5 自动重连验收

验收目标：

1. 中心服务关闭后，Agent 不需要人工重启。
2. Agent 会自动停止旧心跳并进入重连。
3. Agent 按 `3s -> 6s -> 12s -> 24s -> 48s -> 60s -> 60s ...` 的节奏持续重试。
4. 中心服务恢复后，Agent 自动重新连接、重新发送 `hello` 并恢复心跳。

推荐步骤：

1. 先按 `6.3`、`6.4` 启动 Agent，确认日志里已经出现：
   `WebSocket 已连接`
   `WebSocket hello 已确认`
   `WebSocket heartbeat 已确认`
2. 保持手机端 Agent 不关闭，直接停止中心服务。
3. 观察 AutoJs6 日志，预期会先出现类似：
   `WebSocket 连接失败`
   `WebSocket 连接中断，准备第 1 次重连，delay=3000ms`
4. 如果中心持续未恢复，继续观察日志，预期重连等待时间依次变为：
   `6000ms`
   `12000ms`
   `24000ms`
   `48000ms`
   `60000ms`
5. 到达 `60000ms` 之后，后续重连日志仍应继续出现，但等待时间保持 `60000ms` 不再继续增大。
6. 在任意一次重连等待期间，把中心服务重新启动。
7. 继续观察 AutoJs6 日志，预期会依次出现：
   `WebSocket 开始执行第 N 次重连`
   `WebSocket 已连接`
   `WebSocket hello 已确认`
   `WebSocket heartbeat 已确认`
8. 再调用中心设备列表接口，确认对应设备重新回到 `online`。

中心服务恢复后的查询命令：

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:18080/api/v1/devices" | ConvertTo-Json -Depth 8
```

预期结果：

1. 对应设备的 `status` 重新变为 `online`
2. `last_heartbeat_at` 重新开始更新

如果需要重置真机配置，执行：

```bat
.\tools\adb-debug\push-agent.bat http://192.168.1.23:18080 reset
```

或执行 PowerShell 脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\adb-debug\push-agent.ps1 `
  -CenterBaseURL "http://192.168.1.23:18080" `
  -ResetConfig `
  -Run
```

## 8. 常见问题

### 8.1 中心服务不可访问

如果控制台出现连接失败，先确认中心服务是否已启动，并确认端口是否为 `18080`。

真机验收时还需要确认：

1. 手机和电脑在同一个网络内。
2. 中心服务监听地址是 `0.0.0.0:18080`。
3. 手机使用的是电脑局域网 IP，不是 `127.0.0.1`。
4. Windows 防火墙允许手机访问 `18080` 端口。

### 8.2 重复生成 agent_uuid

如果每次启动都生成新的 `agent_uuid`，检查 `/sdcard/脚本/agent/runtime/config.json` 是否被覆盖、是否可以正常写入。

当前 Agent 的规则是：

1. 如果本地配置里已有 `agent_uuid`，始终优先复用。
2. 只有本地没有 `agent_uuid` 时，才会根据手机 `ANDROID_ID` 重新生成一个固定的 `agent_uuid`。
3. 因此正常情况下，同一台手机不应因为重复推送 Agent 代码而注册出新设备。

### 8.3 注册后没有 device_id

检查中心注册接口响应是否为 `status = ok`，并确认响应中是否包含 `data.device_id`。

### 8.4 AutoJs6 没有被拉起

如果 `push-agent.ps1 -Run` 没有成功拉起 AutoJs6，可以：

1. 在 AutoJs6 中手工打开 `/sdcard/脚本/agent/agent.js`。
2. 确认 AutoJs6 包名是否是 `org.autojs.autojs6`。
3. 必要时使用 `-AutoJsComponent` 指定正确的启动组件。

### 8.5 WebSocket 没有连接成功

如果 AutoJs6 日志没有出现 `WebSocket 已连接`，请检查：

1. 中心服务是否使用 `0.0.0.0:18080` 启动。
2. 手机配置中的 `center_base_url` 是否为电脑局域网 IP，例如 `http://192.168.1.23:18080`。
3. Windows 防火墙是否允许手机访问中心服务端口。
4. AutoJs6 日志中是否出现 `WebSocket 连接失败`，并根据错误信息确认网络或地址问题。

### 8.6 中心恢复后没有自动重连

优先检查：

1. `runtime/config.json` 中 `websocket.reconnect_enabled` 是否被错误设置为 `false`
2. AutoJs6 日志中是否已经出现 `准备第 N 次重连`
3. 当前中心服务恢复后，访问地址是否仍与 `center_base_url` 一致
4. 是否存在旧脚本仍在运行，导致你看到的不是最新 Agent 版本日志

## 9. 验收结论

验收时请记录：

1. 验收日期。
2. 中心服务地址。
3. 运行环境：Node.js 或 AutoJs6。
4. `agent_uuid` 是否持久化。
5. `device_id` 是否保存。
6. 中心设备列表是否能看到该设备。
7. 结论：通过、有条件通过或不通过。

## 10. `C2-3` 脚本缺失时自动下载验收

### 10.1 验收目标

本节用于确认：

1. Agent 本地缺少 `shoppe_sync@v0.1.0` 时不会直接失败
2. Agent 会先请求中心脚本清单接口
3. Agent 会再请求中心脚本下载接口
4. 下载完成后会在本机落盘并继续执行任务

### 10.2 前置条件

1. 中心服务已启动，且可访问：
   `http://你的中心地址/api/v1/script/manifest`
   `http://你的中心地址/api/v1/script/download`
2. 手机端 Agent 已连接中心 WebSocket
3. 手机端当前目录中允许你手动删除脚本文件

### 10.3 手动制造“本地脚本缺失”

真机通过 ADB 执行：

```powershell
adb -s 你的设备序列号 shell rm -f /sdcard/脚本/agent/scripts/shoppe_sync/v0.1.0/index.js
adb -s 你的设备序列号 shell rm -f /sdcard/脚本/agent/scripts/shoppe_sync/v0.1.0/manifest.json
```

如果你想先确认文件确实被删除，可以执行：

```powershell
adb -s 你的设备序列号 shell ls /sdcard/脚本/agent/scripts/shoppe_sync/v0.1.0
```
