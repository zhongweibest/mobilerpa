# Agent 侧 4 周快速上线实施清单

> 归档说明：本文档已归档，只作为手机端快速上线早期讨论记录。当前手机端运行契约和上线检查项以 `repos/mobilerpa-agent/docs/手机端运行时最小契约.md` 为准。

本文面向 `repos/mobilerpa-agent` 范围内的快速上线版本，只覆盖 Agent、脚本包、调试工具和真机验收，不包含中心端或前端改造。

## 第 1 周：跑通最小执行链路

本周目标是让一台调试机可以从 Agent 入口拉起一个业务脚本，并产出可追踪的执行日志。

关键交付物：
- Agent 运行时最小契约明确：设备注册、任务拉起、脚本选择、结果回传字段冻结
- 一个可本地调试也可被 Agent 调用的脚本样板
- 真机本地调试流程可复现：推包、启动、看日志、拉截图
- 首个业务脚本的 Happy Path 占位流程可贯通

本周实施清单：
- 明确 Agent 对任务载荷、脚本清单、执行结果的 JSON 结构
- 约定脚本目录标准：`manifest.json`、`src/`、`debug/`
- 增加统一上下文字段：`task_id`、`device_id`、`script_name`、`script_version`、`mode`
- 确定脚本日志最小字段：时间、级别、任务 ID、步骤名、摘要
- 准备 1 台长期在线调试设备和 1 套 ADB 调试说明

完成标准：
- 调试设备上可以启动 Agent 或等价调试入口
- `shoppe` 示例脚本能输出结构化日志并返回成功/失败结果
- 开发人员按文档可在 30 分钟内完成首次真机联调

## 第 2 周：补齐稳定执行与灰度验证能力

本周目标是让 Agent 能稳定执行单任务，并具备失败可定位、版本可区分、脚本可灰度的基本能力。

关键交付物：
- Agent 单任务状态机成型：待执行、执行中、成功、失败、超时、取消
- 脚本版本目录和版本切换规则落地
- Beta 设备组联调流程可执行
- 失败证据链固定下来：日志、截图、页面 dump 至少二选一

本周实施清单：
- 为脚本 `manifest.json` 补全版本兼容字段和依赖约束
- 约定脚本下载或更新后的本地缓存布局
- 明确超时、重试、幂等、取消的处理原则
- 为关键步骤加截图点或页面状态采样点
- 建立至少 5 个核心异常场景的真机回归用例

完成标准：
- 同一业务脚本可以区分 `dev`、`beta`、`prod` 版本
- Agent 执行失败时能定位到脚本版本、步骤名和失败摘要
- Beta 设备组完成一轮带证据的回归验证

## 第 3 周：收敛发布面并建立运维可观测性

本周目标是具备小规模试运行条件，避免上线后“能跑但不可控”。

关键交付物：
- 发布前检查表和回滚原则可执行
- Agent 侧监控字段最小集冻结
- 试运行设备清单与人工兜底流程明确
- 核心业务脚本完成一次端到端试跑

本周实施清单：
- 冻结上线前必查项：设备在线率、脚本版本、权限、网络、存储空间
- 定义任务级指标：领取成功率、启动成功率、执行成功率、平均耗时、超时率
- 明确人工介入条件：连续失败、页面变化、账号风控、系统弹窗
- 准备快速回滚方案：停发新任务、回退脚本版本、切换设备组
- 输出试运行日报模板，按天记录失败原因和修复动作

完成标准：
- 试运行期间每次失败都能归类到脚本、页面、设备、网络中的至少一类
- 发生异常时可在 10 分钟内决定继续灰度、暂停或回滚
- 业务方和执行方对上线门槛达成一致

## 第 4 周：正式上线与首周值守

本周目标是完成快速上线版发布，并用值守机制守住稳定性。

关键交付物：
- `prod` 脚本版本和对应 Agent 兼容范围冻结
- 上线当天值守清单、联系人和升级路径到位
- 首周问题归档机制生效
- 下个迭代的技术债清单沉淀完成

本周实施清单：
- 按设备组逐步放量，避免全量瞬时切换
- 上线当天按小时检查成功率、超时率和异常分布
- 对高频失败问题执行热修复或降级策略
- 收敛文档：已知限制、人工处理方法、版本发布记录
- 整理快速上线版遗留项，进入下一周期排期

完成标准：
- 正式设备组完成上线且具备连续运行记录
- 关键异常能在值守窗口内被识别、止损、复盘
- 首周结束后形成明确的稳定性改进列表

## 跨周持续项

- 真机资源池维护：设备、电量、网络、ADB 权限持续可用
- 页面变更监控：高频页面截图基线持续更新
- 账号与权限管理：测试账号、生产账号、验证码或风控处理流程明确
- 文档同步：调试说明、发布记录、已知问题持续更新

## 当前建议优先级

如果只能先做三件事，优先顺序如下：
- 先冻结 Agent 与脚本之间的最小运行契约
- 再打通一条真机 Happy Path 并保留证据链
- 最后补齐灰度发布与回滚检查表
