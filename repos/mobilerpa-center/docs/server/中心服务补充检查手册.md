# 中心服务补充检查手册

## 1. 目的

本文档不再承担中心端正式主验收入口职责，只用于以下场景：

1. 单独启动中心服务。
2. 页面主线异常时做服务级补充检查。
3. 快速确认健康检查、设备注册和 WebSocket 基础链路是否正常。

中心端正式页面主线验收以 `docs/页面与主线手工验收.md` 为准。

## 2. 前置条件

1. 已安装 Go。
2. 当前目录为 `repos/mobilerpa-center/server`。
3. 已执行过依赖拉取。

## 3. 启动中心服务

在 PowerShell 中执行：

```powershell
$env:CENTER_HTTP_ADDR="127.0.0.1:18080"
$env:CENTER_DB_PATH="D:\dev\code\mobilerpa\.tmp\mobilerpa-center-manual.db"
$env:CENTER_CORS_ALLOWED_ORIGINS="http://localhost:5173,http://127.0.0.1:5173"
$env:CENTER_HEARTBEAT_INTERVAL="30s"
$env:CENTER_DEVICE_OFFLINE_TIMEOUT="90s"
$env:CENTER_DEVICE_OFFLINE_SCAN_INTERVAL="15s"
go run .\cmd\center
```

如需使用本机默认配置，可在 `repos/mobilerpa-center` 目录复制：

```powershell
Copy-Item .\.env.example .\.env
```

## 4. 健康检查

执行：

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:18080/healthz"
```

预期结果：

```json
{
  "status": "ok",
  "service": "mobilerpa-center"
}
```

## 5. 设备注册与设备列表烟测

执行：

```powershell
$body = @{
  agent_uuid  = "agent-test-001"
  device_name = "Pixel Test"
  brand       = "Google"
  model       = "Pixel 8"
  android_id  = "android-test-001"
  adb_serial  = "adb-test-001"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://127.0.0.1:18080/api/v1/device/register" `
  -Method Post `
  -ContentType "application/json" `
  -Body $body
```

预期结果：

1. 返回 `status = ok`。
2. 返回 `device_id`。

再执行：

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:18080/api/v1/devices"
Invoke-RestMethod -Uri "http://127.0.0.1:18080/api/v1/devices/dev_000001"
```

预期结果：

1. 列表中能看到刚注册的设备。
2. 单设备查询能返回正确 `device_id` 与 `agent_uuid`。

## 6. WebSocket `hello/heartbeat` 烟测

如果要快速验收 WebSocket 基础链路，可以执行：

```powershell
cd D:\dev\code\mobilerpa\repos\mobilerpa-center
powershell -ExecutionPolicy Bypass -File .\tools\websocket-smoke-test.ps1
```

预期结果：

1. `hello_ack.payload.status` 为 `ok`。
2. `heartbeat_ack.payload.status` 为 `ok`。
3. `online_device.status` 为 `online`。
4. `offline_device.status` 为 `offline`。

## 7. 页面异常时的补充排查顺序

当 `docs/页面与主线手工验收.md` 中的页面步骤异常时，按下面顺序补查：

1. 先确认中心服务进程仍在运行，且 `healthz` 返回 `ok`。
2. 再确认设备注册、设备列表、单设备查询三条基础接口是否可用。
3. 如果页面依赖设备在线状态，补做一次 WebSocket `hello/heartbeat` 烟测。
4. 如果问题涉及 Agent 下发或脚本执行，再查看 `repos/mobilerpa-agent/docs/手机端手工验收.md`。
