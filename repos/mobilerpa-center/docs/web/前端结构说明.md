# MobileRPA Center Web 最小结构说明

## 1. 工程骨架

`web/` 目录已经形成可持续扩展的管理后台工程骨架。这里的管理后台不是独立于中心服务之外的另一套业务中心，而是中心端的 Web 管理入口。包含：

- `Vue 3 + Vite + TypeScript` 基础工程文件
- `src/router/` 路由入口
- `src/layouts/` 后台骨架布局
- `src/api/` 接口请求层
- `src/stores/` Pinia 状态层
- `src/pages/home/` 主页看板占位页
- `src/pages/devices/` 设备列表页入口
- `src/styles/` 页面全局样式

## 2. 目录结构

最小结构如下：

```text
web/
  README.md
  docs/
    前端结构说明.md
    archive/
      前端快速上线清单.md
  src/
    api/
    layouts/
    pages/
      home/
      devices/
    router/
    stores/
    styles/
    types/
    utils/
  index.html
  package.json
  tsconfig.json
  vite.config.ts
```

可按需补充：

- `pages/workflows/`
- `pages/bindings/`
- `components/`
- `public/`

## 3. 页面范围

管理后台正式覆盖以下页面或入口：

- 设备列表页
- 设备绑定页或绑定操作入口，基于 `location_nodes` 位置树节点
- 计划任务页
- 计划任务实例页
- 工作流编排页
- 日志与截图查看入口

可保留入口但不作为完整主线能力的页面：

- 脚本版本管理

## 4. 模块职责

- `api/`: 只放接口请求和 DTO 转换，不放页面状态
- `pages/`: 按业务页面拆分，避免把所有页面堆到一个目录
- `stores/`: 只放跨页面共享状态，如用户信息、筛选缓存、全局字典
- `types/`: 放设备、任务、日志、截图等核心类型
- `utils/`: 放时间格式化、状态映射、查询参数处理等通用逻辑

## 5. 联调约定

为了减少前后端互相阻塞，统一按下面约定实现：

- 列表字段缺失时以前端降级展示为主，不阻塞页面骨架联调
- 状态枚举统一在 `types/` 和 `utils/` 中集中映射
- 所有时间字段统一约定时区与格式
- 所有空值字段统一约定默认展示文案

## 6. 实现侧重点

前端实现时应优先保证以下主线能力：

1. 基础工程、路由和后台壳层布局稳定。
2. 设备列表、计划任务、计划任务实例三个正式运行入口可直接使用。
3. 工作流编排、日志、截图、绑定入口与位置树节点维护能支撑真实联调和排障。

脚本版本管理等非主线页面可以保留入口，但不应影响正式运行链路的可用性。

## 7. 能力边界

管理后台前端优先满足：

1. 能看到设备状态。
2. 能发起和查看基础计划任务与工作流运行。
3. 能查看日志、结果和截图等排障信息。
4. 能支撑 10 到 30 台设备的小规模试运行。

后台壳层补充约定：

1. 左侧功能菜单作为统一导航入口长期保留。
2. 首页默认保留为“主页看板”占位页，先不堆砌假数据。
3. 设备列表通过左侧菜单进入，不再直接占用首页路由。
4. 非主线能力不再保留占位菜单入口，避免影响正式使用与验收。

暂不追求：

1. 完整权限体系。
2. 大规模灰度发布能力。
3. 复杂交互体验打磨。

## 8. 对后端的最小依赖

前端要稳定运行，至少需要以下接口与字段约定：

1. 设备列表接口与字段定义。
2. 计划任务列表与计划任务实例列表接口定义。
3. 创建计划任务接口入参。
4. 计划任务与工作流运行状态枚举。
5. 设备在线状态判定规则。
6. 日志与截图查看接口的最小返回结构。
7. 工作流定义接口，以及计划任务/计划任务实例查询与停止接口。

如果这些字段频繁变化，页面结构、状态映射和交互会持续返工，因此应尽量保持稳定。

## 9. 工作流与计划任务入口约定

- “工作流编排”菜单只负责维护工作流定义，不再作为正式运行态主入口。
- 正式运行入口统一收口到“计划任务”和“计划任务实例”两个页面。
- 如果需要运行一个工作流，应先在“计划任务”页创建 `target_type=workflow` 的计划任务，再在“计划任务实例”页查看运行状态、事件、追加设备和移除设备。
- `schedule_type=once` 的前端语义固定为“一次性（手工启动）”，不表示指定年月日时分秒自动执行。
- `schedule_type=daily` 的前端语义固定为“按天循环（自动调度）”；页面中的“立即启动”只表示人工立刻拉起一个实例，不改变后续 daily 自动调度行为。
- “工作流实例”页面已下线，不再保留历史兼容查看入口。
- 工作流后端接口和计划任务后端接口的手工验收统一收口到 `docs/server/中心服务手工验收.md`。

## 10. 任务中心前端收口约定
- 前端左侧导航不再保留“任务中心”正式入口。
- 正式运行入口统一收口到“计划任务”“计划任务实例”“工作流编排”。
- 设备页不再提供“人工结束手工任务”能力，前端正式视角仅展示计划任务实例占用。
- 若后端保留历史手工任务兼容接口，其使用范围也不进入前端正式页面。
