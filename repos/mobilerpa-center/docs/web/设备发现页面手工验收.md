# 设备发现页面手工验收

## 1. 目标

本文档用于验收前端“设备发现”页面是否具备以下能力：

1. 左侧菜单可进入“设备发现”页面。
2. 页面可展示局域网无线调试设备列表。
3. 页面支持多选设备。
4. 页面支持填写中心地址并触发批量下发 Agent。
5. 页面支持查看逐设备下发结果。
6. 页面支持对单台设备执行“启动 Agent”和“停止 Agent”。

## 2. 前置条件

1. 中心服务已启动，并正确配置 `CENTER_ADB_PATH`、`CENTER_AGENT_ROOT_PATH`、`CENTER_TOOLKIT_PATH`、`CENTER_BASE_URL`。
2. 前端开发环境已启动。
3. `web/.env.development` 中的 `VITE_API_PROXY_TARGET` 已指向真实中心服务地址。
4. 局域网内至少有一台已完成无线调试配对的手机。

## 3. 打开页面

浏览器访问：

```text
http://127.0.0.1:5173/discovery
```

或先访问：

```text
http://127.0.0.1:5173
```

再通过左侧菜单点击“设备发现”进入。

## 4. 页面验收步骤

### 4.1 菜单与页面骨架

预期结果：

1. 左侧菜单中存在“设备发现”入口。
2. 页面标题显示“设备发现”。
3. 页面上方存在“刷新发现结果”按钮。
4. 页面中能看到“中心地址”“重置设备已有配置”“推送后自动运行 Agent”三个操作项。

### 4.2 刷新发现结果

操作：

1. 点击“刷新发现结果”按钮。

预期结果：

1. 页面发起 `GET /api/v1/discovery/devices` 请求。
2. 页面表格显示发现到的设备。
3. 每行至少展示设备名、中心设备 ID、ADB 地址、来源、类型、连接状态。

### 4.3 多选设备

操作：

1. 勾选一台或多台设备。
2. 尝试使用表头复选框全选。

预期结果：

1. 只有 `connectable=true` 的设备允许勾选。
2. “下发并启动 Agent（N）”或“仅下发 Agent（N）”中的数量会随勾选变化。
3. 表头复选框可一次性勾选所有可连接设备。

### 4.4 批量下发

操作：

1. 在“中心地址”输入框填写真机可访问的中心服务地址，例如 `http://192.168.0.155:18080`。
2. 根据需要决定是否勾选“重置设备已有配置”。
3. 根据需要决定是否勾选“推送后自动运行 Agent”。
4. 点击批量下发按钮。

预期结果：

1. 页面发起 `POST /api/v1/discovery/agent-deployments` 请求。
2. 请求体中不需要包含 `script_name` 和 `script_version`。
3. 按钮进入“下发中...”状态。
4. 页面出现“下发结果”表格。
5. 结果表格按设备逐项展示连接、推送、启动、状态和说明。

请求体示例：

```json
{
  "adb_endpoints": [
    "192.168.0.120:37123",
    "192.168.0.121:38741"
  ],
  "center_base_url": "http://192.168.0.155:18080",
  "reset_config": false,
  "run_agent": true
}
```
