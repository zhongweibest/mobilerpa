# MobileRPA Center Web 4 周快速上线实施清单

> 归档说明：本文档已归档，只作为前端快速上线早期讨论记录。当前前端范围、节奏和依赖以 `repos/mobilerpa-center/docs/web/前端结构说明.md` 为准。

## 1. 目标边界

本清单只覆盖 `repos/mobilerpa-center/web` 的后台前端部分，目标是在 4 周内支撑早期生产版上线，优先满足：

- 能看到设备状态
- 能发起和查看基础任务
- 能查看日志、结果、截图等排障信息
- 能支撑 10 到 30 台设备的小规模试运行

当前阶段暂不把前端目标放在：

- 完整权限体系
- 完整报表中心
- 完整异常中心
- 大规模灰度发布能力
- 复杂交互体验打磨

## 2. 周交付物

### 第 1 周：前端起盘与联调骨架

关键交付物：

- 初始化 Vue 3 + Vite + TypeScript 管理后台工程
- 建立基础目录结构：`src/app`、`src/router`、`src/layouts`、`src/pages`、`src/api`、`src/stores`、`src/types`
- 接入 Vue Router、Pinia、Element Plus
- 完成基础布局框架：登录占位、顶部导航、侧边菜单、内容区、404 页
- 建立环境配置约定：开发环境 API Base URL、WebSocket Base URL
- 输出接口占位层：设备列表、任务列表、任务创建、日志查询、结果查询
- 与后端对齐第一版字段协议，至少冻结设备列表和任务列表的最小字段

完成标准：

- 前端项目可本地启动
- 页面路由可切换
- 假数据下能展示设备列表和任务列表骨架
- 前端已明确需要后端提供的最小接口与字段

### 第 2 周：设备页与任务页最小闭环

关键交付物：

- 完成设备列表页
- 完成设备详情抽屉或详情页最小版
- 展示在线状态、最后心跳、当前任务、绑定状态、物理槽位等基础信息
- 完成任务列表页
- 完成单任务创建与下发表单
- 完成基础状态映射：待执行、执行中、成功、失败、取消
- 建立统一错误提示、空状态、加载状态组件

完成标准：

- 可以通过真实接口查看设备列表
- 可以手工创建并下发一个基础任务
- 可以从页面看到任务状态变化
- 页面具备基本可用的加载、失败、空数据反馈

### 第 3 周：排障链路与试运行支撑

关键交付物：

- 完成任务详情页或任务结果侧栏
- 接入基础日志查看能力
- 接入截图预览能力
- 完成待绑定设备页或设备绑定操作入口
- 支持按设备、状态、时间进行基础筛选
- 补齐联调所需前端文档：字段说明、状态流转、页面入口
- 整理 3 到 10 台设备联调问题并快速修复前端阻塞项

完成标准：

- 运营或测试可通过页面定位某台设备当前执行情况
- 失败任务至少能看到结果、错误摘要、基础日志、截图入口
- 设备绑定与任务查看链路可贯通

### 第 4 周：上线收口与可操作性补强

关键交付物：

- 补齐高频操作细节：刷新、筛选保留、最近操作提示、失败重试提示
- 补齐前端上线前检查项：环境变量、构建命令、部署产物说明、反向代理要求
- 完成 10 到 30 台试运行期间的前端问题修复
- 补齐最小操作手册：如何看设备、如何发任务、如何定位失败
- 与后端共同收口接口字段，减少上线后再改协议的风险
- 确认首版不做的内容并显式标注占位

完成标准：

- 前端可构建并交付部署
- 非研发角色可按文档完成基础操作
- 试运行期间主要问题可通过后台页面完成初步定位

## 3. 前端优先级顺序

建议严格按以下顺序推进：

1. 工程骨架
2. 设备页
3. 任务页
4. 任务结果与日志截图
5. 设备绑定入口
6. 版本管理占位
7. 异常中心和报表占位

原因是第 1 版后台的核心目标不是“功能丰富”，而是“能看、能发、能定位”。

## 4. 前端对后端的最小依赖

若要在 4 周内快速上线，前端最晚应在第 1 周内拿到以下稳定内容：

- 设备列表接口与字段定义
- 任务列表接口与字段定义
- 创建任务接口入参
- 任务状态枚举
- 设备在线状态判定规则
- 日志与截图查看接口的最小返回结构

如果这些字段在第 2 周仍频繁变化，前端页面会反复返工，直接影响上线节奏。
