# mobilerpa-center

这是 MobileRPA 的中心控制仓库，负责设备管理、任务调度、脚本版本管理以及后台管理接口。

## 目录说明
- `server/`: Go 后端服务
- `web/`: Vue 管理前端
- `docs/server/`: 中心服务架构、开发约定和手工验收文档
- `docs/web/`: 中心前端结构说明和手工验收文档

## 仓库职责
本仓库负责以下内容：
- 设备注册与绑定
- HTTP 与 WebSocket 服务
- 任务编排与调度
- 脚本版本元数据管理
- 日报与统计聚合
- 面向后台的管理接口

## 文档语言约定
本仓库中的设计、规范、接口、部署、运维文档默认使用中文。

## 调度约定
- 计划任务默认采用轻量扫描协程 + 异步执行 worker 的方式运行。
- 涉及多设备批量启动、停止、下发的功能，优先使用受控并发模型，不直接在扫描线程中串行执行。

## 主键约定
- 中心服务所有核心业务主键默认直接采用数据库自增整数主键。
- `device_id`、`task_id`、`workflow_def_id`、`workflow_instance_id`、`workflow_run_id`、`plan_def_id`、`plan_run_id` 等字段如果作为主标识存在，应直接设计成自增整数类型，而不是带前缀的字符串编号。
- 新增数据表时，禁止再使用 `dev_000001`、`task_000001`、`wf_000001` 这类前缀字符串作为主键方案。
- 默认排序优先按各表自增主键倒序处理，前端页面默认不展示主键时，也应遵循该排序规则。
