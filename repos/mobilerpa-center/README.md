# mobilerpa-center

这是 MobileRPA 的中心端仓库。中心端由 Go 中心服务后端和 Vue 管理后台前端共同组成，统一承担设备管理、设备发现、计划任务调度、脚本管理、工作流定义、Agent 管理与调度等平台能力；管理后台不是独立于中心服务之外的另一套业务中心，而是中心端的 Web 管理入口。

## 目录说明
- `server/`: Go 后端服务
- `web/`: Vue 管理前端
- `docs/server/`: 中心服务架构、开发约定和手工验收文档
- `docs/web/`: 中心前端结构说明和手工验收文档

## 仓库职责
本仓库负责以下内容：
- 设备注册与绑定
- 设备发现与 Agent 下发
- HTTP 与 WebSocket 服务
- 计划任务编排与调度
- 脚本版本元数据管理
- 工作流定义维护
- 面向管理后台的接口与前端实现

## 文档语言约定
本仓库中的设计、规范、接口、部署、运维文档默认使用中文。

## 调度约定
- 计划任务默认采用轻量扫描协程 + 异步执行 worker 的方式运行。
- 涉及多设备批量启动、停止、下发的功能，优先使用受控并发模型，不直接在扫描线程中串行执行。

## 运行入口约定
- 正式运行入口统一收口到计划任务，不再继续扩展“任务中心”作为主线功能。
- 如果后续要下线任务中心，只能先下线 manual 入口和前端页面，不能在计划任务、工作流或 Agent 协议仍依赖 `task` 执行底座时直接删除底层 `task` 域代码。
- 工作流实例层已经移除，正式运行态统一为 `plan_def -> plan_run -> plan_device_run`。
- 公开 `/api/v1/tasks` 已从正式路由移除；`internal/task` 仅作为脚本型计划任务执行底座保留。
- 工作流正式执行链路默认使用 `workflow_session_*` 协议族，由中心下发会话、由 Agent 本地推进状态机。

## 主键约定
- 中心服务所有核心业务主键默认直接采用数据库自增整数主键。
- `device_id`、`task_id`、`workflow_def_id`、`plan_def_id`、`plan_run_id` 等字段如果作为主标识存在，应直接设计成自增整数类型，而不是带前缀的字符串编号。
- 新增数据表时，禁止再使用 `dev_000001`、`task_000001`、`wf_000001` 这类前缀字符串作为主键方案。
- 默认排序优先按各表自增主键倒序处理，前端页面默认不展示主键时，也应遵循该排序规则。
